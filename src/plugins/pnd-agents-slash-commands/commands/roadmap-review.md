# /roadmap-review

Analyze product roadmaps and OKRs to identify risks, missing dependencies, timeline issues, and improvement opportunities.

## Usage

```
/roadmap-review [paste roadmap content or OKRs to review]
```

## Examples

```
/roadmap-review [paste Q1 roadmap here]
/roadmap-review Analyze the 2024 product OKRs for feasibility
/roadmap-review Review the mobile app launch timeline
```

## What This Command Does

1. **Analyzes** roadmap structure and timelines
2. **Identifies** missing dependencies
3. **Assesses** timeline realism
4. **Finds** resource conflicts
5. **Evaluates** OKR alignment
6. **Suggests** improvements

## Output Format

The agent provides:
- Roadmap health assessment
- Risk inventory with severity
- Missing dependency list
- Timeline feasibility analysis
- Resource conflict identification
- OKR alignment check
- Specific improvement recommendations

## Analysis Categories

- **Dependencies**: Technical, team, external
- **Timeline**: Realistic vs. aggressive
- **Resources**: Capacity and skills gaps
- **Risks**: Market, technical, organizational
- **Alignment**: Strategy and OKR fit

## Risk Severity Levels

- **Critical**: Roadmap blockers
- **High**: Significant impact likely
- **Medium**: Manageable with attention
- **Low**: Minor concerns

## Review Deliverables

- Executive risk summary
- Detailed findings report
- Recommended adjustments
- Alternative timeline options
- Success criteria validation

## Pandora Planning Standards

- Quarterly planning alignment
- Cross-team dependency mapping
- Buffer time for unknowns
- Clear success metrics
- Regular checkpoint reviews
