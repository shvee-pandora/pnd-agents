# /analytics

Generate comprehensive sprint reports, velocity metrics, and team performance analytics from Jira data.

## Usage

```
/analytics [describe the report or metrics needed]
```

## Examples

```
/analytics Generate a sprint velocity report for the last 6 sprints
/analytics Create a burndown analysis for Sprint 42
/analytics Show team capacity utilization for Q4
```

## What This Command Does

1. **Analyzes** sprint and project data
2. **Calculates** velocity and throughput metrics
3. **Generates** burndown/burnup charts data
4. **Identifies** bottlenecks and blockers
5. **Produces** executive-ready reports
6. **Suggests** process improvements

## Output Format

The agent provides:
- Sprint summary with key metrics
- Velocity trend analysis
- Burndown chart data
- Cycle time distribution
- Blocker analysis
- Recommendations for improvement

## Metrics Calculated

- **Velocity**: Story points completed per sprint
- **Throughput**: Issues completed per time period
- **Cycle Time**: Time from start to completion
- **Lead Time**: Time from creation to completion
- **WIP**: Work in progress at any time
- **Escaped Defects**: Bugs found post-release

## Report Types

- Sprint retrospective summary
- Release readiness report
- Team capacity analysis
- Quality metrics dashboard
- Predictability assessment

## Pandora Analytics Standards

- Data-driven insights only
- Clear visualizable metrics
- Actionable recommendations
- Historical trend comparison
- Risk-adjusted forecasting
